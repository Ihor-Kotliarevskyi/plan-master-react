# UI Guide

## Огляд

Plan Master має чотири основні вкладки:

- `Гант`
- `Фінанси`
- `Контрагенти`
- `Аналітика`

Інтерфейс побудований навколо одного HTML shell (`index.html`) і набору overlay/modal-вікон. Більшість повторно вживаних контролів стилізовано через `css/controls.css`.

## Гант

### Верхня панель

Поточний toolbar вкладки `Гант` містить:

- пошук по назві роботи
- multi-filter за контрагентом
- multi-filter за станом оплат
- toggle критичного шляху
- toggle SVG-стрілок залежностей
- кнопку списку залежностей
- toggle групування за категоріями
- кнопку повторного відкриття банера прострочених робіт
- zoom in / zoom out
- toggle монохромного режиму барів
- вибір кольору барів у монохромному режимі

### Таблиця Ганта

Поточна таблиця підтримує:

- групування рядків за категоріями
- drag-and-drop порядку робіт
- resize/drag барів
- baseline overlay
- підсвічування сьогоднішнього тижня
- підсвічування ланцюжка залежностей по кліку на бар
- приглушення нерелевантних рядків під час пошуку або chain highlight

### Банер прострочених робіт

Коли є незавершені роботи з датою завершення в минулому, показується `overdue-banner`:

- кількість прострочених робіт
- короткий preview перших рядків
- розгортання списку
- закриття з можливістю повторно відкрити з toolbar

## Модалка роботи

### Загальна структура

Основне вікно редагування роботи відкривається через `js/modal.js` і має дві внутрішні вкладки:

- `Загальне`
- `Кошторис`

### Вкладка `Загальне`

Фактичні секції:

- назва роботи
- категорія
- дати початку/завершення
- прогрес
- фази
- залежності
- міні-граф залежностей
- бюджет / витрати / автоперерахунок

### Фази

Підтримується список фаз з:

- окремими датами початку/завершення
- окремим прогресом кожної фази
- автоматичним підрахунком загального прогресу
- блокуванням наступних фаз, якщо попередня ще не завершена

### Залежності

У модалці роботи залежності редагуються як chips + type editor:

- вибір попередника зі списку існуючих робіт
- типи `FS`, `SS`, `FF`
- `% threshold` для `SS`
- міні-мережа попередників і наступників поточної роботи

### Вкладка `Кошторис`

Кошторис відкривається прямо всередині модалки роботи, а не окремим великим екраном.

Підтримуються:

- позиції `material`, `work`, `equipment`, `other`
- постачальник / контрагент
- кількість, одиниця, ціна
- договір і дата договору
- акти
- платежі
- автопідрахунок бюджету та фактичних витрат

## Список залежностей

Окрема overlay-модалка `dep-list-modal` показує:

- всі залежності проєкту
- фільтри `all / FS / SS / FF`
- колонки: попередник, тип, наступник, критичність
- перехід до gantt з підсвічуванням ланцюжка після кліку

## Нотатки

Модалка нотаток підтримує:

- список нотаток до роботи
- додавання нової нотатки
- редагування
- soft-delete
- історію змін нотатки

## Керування проєктами

### Налаштування проєкту

Модалка налаштувань проєкту дозволяє:

- змінити назву
- змінити стартовий місяць
- змінити стартовий рік
- змінити тривалість у місяцях

При зміщенні старту проєкту зміщуються й дати задач/фаз.

### Менеджер проєктів

Поточний менеджер проєктів підтримує:

- список локально доступних проєктів
- перейменування
- створення нового
- видалення
- завантаження демо-проєкту

## Кабінет користувача

### Що є зараз

`user-modal` об’єднує:

- дефолтні параметри нових проєктів
- toggle теми
- baseline controls
- avatar upload / clear
- редагування імені
- auth flow для Supabase
- sync/logout стан

### Ліва колонка

- стартовий місяць для нових проєктів
- стартовий рік
- тривалість
- тема
- блок baseline:
  - збереження baseline
  - перезапис baseline
  - видалення baseline
  - показ / приховування baseline

### Права колонка

- великий avatar preview
- ім’я та email
- завантаження/очищення аватара
- редагування імені
- блок акаунта:
  - login/register для гостя
  - email, sync state і logout для залогіненого користувача

## Фінанси

### Режими

Вкладка `Фінанси` має два режими:

- `Графік`
- `Таблиця`

Активний режим зберігається в UI state.

### Фільтри

Поточна панель фільтрів містить:

- пошук
- multi-filter за категоріями
- multi-filter за статусами
- multi-filter за контрагентами
- budget min / max
- `тільки з бюджетом`
- reset filters
- toggle EVM
- bulk delete видимих робіт

### Графіки

Поточний overview включає:

- S-криву
- toggle тижневих витрат
- окремий weekly cost chart
- зміну висоти графічного блоку

Після поточної дати фактичні ряди показуються як прогнозні ділянки.

### Таблиця фінансів

Поточна таблиця підтримує:

- sticky header
- сортування
- resize колонок
- пошук і фільтри спільні з графіками

## Контрагенти

### Загальна логіка

Вкладка `Контрагенти` не працює з окремим масивом contractors. Вона агрегує дані з `task.costItems[].supplier`.

### Що є у вкладці

- пошук
- фільтри
- contractor tools menu
- основна таблиця контрагентів
- розкриття платежів по контрагенту
- ручне додавання/редагування позицій і платежів
- імпорт `.xlsx` / `.xls` / `.csv`
- реєстри платежів
- експорт/друк реєстрів

### Реєстри платежів

Окрема modal/overlay для реєстрів платежів підтримує:

- формування поточного реєстру з активних фільтрів
- збереження іменованого реєстру
- друк
- експорт `xlsx`
- експорт `csv`
- видалення збереженого реєстру

## Друк і PDF

### Print dialog

Поточна `print-modal` має дві частини:

- панель параметрів друку
- preview panel

### Можливості

- вибір секцій для друку
- вибір графіків
- paper / orientation / fit / scale / quality / margin
- page preview
- paging preview вперед/назад
- друк через browser print
- PDF export через `html2canvas` + `jsPDF`

### Що потрапляє в друк

За поточним кодом друк/експорт може включати:

- gantt
- finance summary
- S-curve
- weekly cost chart
- charts pane

## Спільний доступ і readonly

### Що бачить користувач

Коли проєкт відкрито з роллю `viewer`:

- показується `readonly-banner`
- блокується взаємодія з gantt
- приховується кнопка додавання роботи

### Share UI

Кнопка `share-btn` відкриває модалку доступів:

- список поточних share-записів
- вибір ролі `viewer / editor / admin`
- видалення share
- додавання нового користувача за email

## Аналітика

Вкладка `Аналітика` використовує `js/charts.js` і показує auto/custom charts. Вона також використовується print pipeline для експорту вибраних графіків.

## Спільні стилі контролів

Базові UI-класи:

- `.btn`
- `.btn-sm`
- `.btn-acc`
- `.btn-danger`
- `.btn-tog`
- `.fg`

`css/controls.css` залишається основним шаром для:

- buttons
- inputs
- selects
- textarea
- пошукових полів
- filter panels
- focus/hover/disabled states
