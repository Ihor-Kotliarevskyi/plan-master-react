# Plan Master

Сучасний веб-застосунок для управління проєктами за допомогою інтерактивних діаграм Ганта. Працює як PWA з офлайн-доступом, хмарною синхронізацією та підтримкою аналізу освоєного обсягу (EVM).

## Ключові Особливості

### 📊 Діаграма Ганта
- **Завдання з фазами** — розбивайте роботи на підетапи з незалежними термінами та прогресом
- **Залежності між завданнями** — підтримка типів FS (Finish-Start), SS (Start-Start), FF (Finish-Finish)
- **Critical Path Method (CPM)** — автоматичне виділення критичного шляху
- **Візуалізація залежностей** — інтерактивні SVG-стрілки з підсвічуванням ланцюжків
- **Baseline (план/факт)** — зберігання знімка для порівняння
- **Drag-and-drop** — редагування термінів прямо на діаграмі
- **Категоризація** — кольорова кодифікація та групування за категоріями
- **Режим перегляду** — спільний доступ без можливості редагування

### ☁️ Хмарна синхронізація
- **Реєстрація та вхід** — через Supabase Auth
- **Надійна синхронізація** — використання версіонування (`_localVersion` vs `_serverVersion`) для запобігання втраті даних
- **Write-Ahead Buffer** — всі зміни миттєво зберігаються в локальний буфер `gantt_buf` перед відправкою на сервер
- **Автосинхронізація** — інтелектуальний debounce 800 мс з урахуванням перемикання проєктів
- **Спільний доступ** — ролі owner, manager, editor, viewer з контролем прав на рівні БД (RLS)
- **Офлайн-режим** — повна функціональність при відсутності інтернету з автоматичним PUSH при відновленні зв'язку


### 💰 Управління фінансами
- **Кошторис по завданням** — матеріали, роботи, техніка, інше
- **Платежі** — авансові, акти, рахунки з датами та примітками
- **EVM-метрики** — BCWP, ACWP, BAC, CPI, EAC, ETC, VAC
- **S-крива бюджету** — кумулятивна крива освоєння з фільтрацією за категоріями, статусами, контрагентами, бюджетом і пошуком
- **Прогноз після поточної дати** — фактична крива та фактичні стовпчики після today відображаються пунктиром як прогноз, а не як підтверджений факт
- **Фінансова таблиця** — сортування, пошук по роботах/кошторису/контрагентах/платежах, зміна ширини колонок

### 🤝 Контрагенти та платежі
- **Єдина таблиця контрагентів** — агрегує постачальників з кошторису по всіх роботах
- **Розгортання контрагента** — показує окремі платежі з сортуванням по колонках
- **Ручне додавання** — додавання контрагентів, позицій кошторису та оплат
- **Редагування та видалення** — редагування позицій і платежів без зміни самого контрагента в модалках редагування
- **Імпорт Excel/CSV** — імпорт таблиць контрагентів, позицій кошторису та оплат, а також експорт шаблону
- **Реєстри платежів** — формування, збереження, друк, експорт у XLSX/CSV
- **Нерозподілений кошторис** — окремий рядок для прогнозних сум робіт, які ще не рознесені по контрагентах

### 📈 Аналітика
- **Конструктор графіків** — бар, лінія, кругові діаграми
- **Авто-графіки** — статуси завдань, розподіл по категоріях, витрати
- **Експорт** — Excel (XLSX), PDF з форматуванням

### 🎨 Дизайн і доступність
- **Темна/світла тема** — переключення в режимі реального часу
- **Адаптивні модальні вікна** — фіксовані розміри з прокруткою контенту
- **Consistent UI** — двоколонні лейаути, стилізовані форми
- **Unified controls** — спільний CSS-шар для кнопок, інпутів, селектів, пошуку, тулбарів і focus/hover станів
- **Keyboard shortcuts** — інтуїтивна навігація
- **Mobile-friendly** — PWA для мобільних пристроїв

## Технологічний Стек

- **Frontend:** Vanilla JavaScript (без фреймворків)
- **Database:** Supabase (PostgreSQL, Row Level Security)
- **Authentication:** Supabase Auth
- **Visualization:** Chart.js 4.x, SVG (залежності)
- **Icons:** Lucide Icons
- **Export:** SheetJS (Excel), jsPDF + html2canvas (PDF)
- **PWA:** Service Worker, Offline Cache
- **UI Components:** SweetAlert2, Custom HTML5 controls

## Запуск

Застосунок — статичний SPA без етапу збирання. Відкрийте через локальний HTTP-сервер:

```bash
# Python 3
python -m http.server 8080

# Node.js
npx serve .

# VS Code: Live Server extension
```

Відкрийте `http://localhost:8080` у браузері.

> Примітка: Service Worker та деякі Web API не працюють при відкритті через `file://`

## PWA та Офлайн

Service Worker реєструється при першому завантаженні. Поточна кеш-версія: `gantt-pro-v9`.

**Кешовані ресурси:**
- `index.html`, `manifest.json`
- CSS та JS файли
- Chart.js CDN
- Lucide Icons CDN

**Стратегія:** network-first для API запитів з fallback на кеш.

## Особливості Інтерфейсу

### Модальні Вікна

Усі модальні вікна спроектовані з фіксованими розмірами, двоколонними лейаутами та прокруткою контенту:

- **Task Modal** (1080×780px) — створення/редагування завдання з фазами, залежностями, кошторисом
- **User Cabinet** (800×580px) — параметри користувача, теми, хмарний акаунт
- **Dependency List** (680×600px) — таблиця залежностей з фільтрацією
- **Costs Modal** — вкладка кошторису всередині модалки роботи; заголовок показує назву роботи
- **Contractor Entry Modal** — ручне додавання контрагента/позиції/платежу; у режимі додавання платежу контрагент зафіксований
- **Payment Register Modal** — поточний реєстр за фільтрами та список збережених реєстрів
- **Notes Modal** — нотатки до завдань

### Залежності

Візуалізація залежностей через SVG-стрілки з支持:
- **Типи залежностей** — FS, SS, FF з кольоровою розрізненням
- **Chain Highlighting** — виділення всього ланцюжка залежностей при кліку
- **Performance** — автоматичне приховування стрілок при >30 завданнях
- **Critical Path** — червона підсвітлення для критичних залежностей

### Теми

Повна підтримка світлої та темної тем через CSS-змінні:

```css
--surf        /* основний фон */
--surf2       /* вторинний фон */
--bord        /* кордони */
--acc         /* акцент (синій) */
--txt         /* основний текст */
--txt2        /* вторинний текст */
```

Переключення через кнопку в кабінеті користувача. Вибір зберігається в localStorage та синхронізується з хмарою.

## Розробка

### Додання нового модального вікна

1. Додайте HTML-структуру в `index.html`
2. Створіть стилі в `css/features.css` (розмір, позиціонування)
3. Додайте стилі для бібліотеки в `css/modal.css` (форми, кнопки)
4. Напишіть логіку в новому файлі `js/modal-*.js`
5. Викличте `lucide.createIcons()` після рендеру для ініціалізації іконок

### Редагування тем

CSS-змінні визначені в `css/variables.css`. Темна тема перевизначена в `css/theme.css` під селектором `[data-theme="dark"]`.

Приклад додання нової змінної:

```css
/* variables.css */
--new-color: #ffffff;

/* theme.css */
[data-theme="dark"] {
  --new-color: #1a1a1a;
}
```
